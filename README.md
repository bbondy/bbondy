## About
I am a software developer focused on systems and product engineering across C++, Rust, Go, Python, C#, and JavaScript.
I co-founded Brave in 2015 with Brendan Eich and currently serve as CTO and board member.

Outside work, I spend time with family and run long distances, often while listening to audiobooks.

## Current Focus
- Privacy-preserving browser and AI features at Brave
- Developer tooling (Rust + CLI)
- Endurance training and backyard ultra prep

## Selected Projects
- [Brave macOS, Windows, and Linux browser](https://github.com/brave/brave-core): Block ads and trackers, saves data, and speeds up web browsing.
- [Brave Wallet](https://github.com/brave/brave-core): Secure, native, built-in crypto wallet that supercharges Brave browser for Web3.
- [Brave Leo](https://github.com/brave/brave-core): AI-powered assistant built into the Brave browser, offering features like question answering, content summarization, and more—all while maintaining privacy.
- [Guardrails](https://github.com/bbondy/guardrails): Native Rust CLI wrapper that blocks unsafe or prompt-injection style output from other CLIs, with both blocking (check) and sanitizing (filter) modes.
- [Macro Lap](https://github.com/bbondy/macro-lap): Macro Lap lets you track a larger macro lap without losing the normal lap button. Double-tap the lap button to start or end a macro lap while single taps still record regular laps. Perfect for events like backyard ultras where you want stats for the full loop and the smaller splits within it.
- [IPFS in Brave](https://github.com/brave/brave-core): Deep integration of InterPlanetary File System (IPFS) support in Brave browser, allowing users to access content via content hashes (CIDs) instead of URLs. This enables content distribution without high bandwidth costs, data deduplication, and access to content even if it has been censored. Users can choose between using a local IPFS node or public gateways.
- [Link Bubble Browser](https://github.com/brave/link-bubble): Link Bubble makes the Android mobile web more efficient. When you click on a link in any app, Link Bubble loads that webpage in the background and presents it in a floating bubble, leaving you free to continue using your current app. Unique amongst Android web browsers, it saves precious time normally spent watching pages load.
- [brianbondy.com](https://github.com/bbondy/go-brianbondy): The current website for brianbondy.com. This website is written in Go. The version before that was written in Node, Hapi, Redis React, and ES6 transpiled with Babel. Before that the previous versions were written in Google App Engine, Django, ASP .NET (C#), Flash / ActionScript, and the original one was generated from C++ code.
- [Brave Ad Block Engine (C++)](https://github.com/brave/adblock-rust): Adblock Plus filter parsing and matching for lists like EasyList. This library allows users to prevent page elements, such as advertisements, from being downloaded and displayed. It is an implementation of AdBlockPlus' filter rules. This library is in no way affiliated with Adblock Plus. It just parses the same rules as Adblock Plus does. It does it significantly faster than other adblock implementations.
- [Babel JSM module formatter](https://github.com/bbondy/babel-jsm-plugin): Babel plugin for converting ES6 modules into JavaScript code modules (JSM). JavaScript code modules let multiple privileged JavaScript scopes share code. For example, a module could be used by Firefox itself as well as by extensions, in order to avoid code duplication.
- [Simple AWS EC2 wrapper library](https://github.com/bbondy/simple-ec2): Simple AWS SDK wrapper library for starting, stopping, listing, and executing commands on Amazon Elastic Compute Cloud (EC2)instances. EC2 is a central part of Amazon.com's cloud computing platform, Amazon Web Services (AWS). EC2 allows users to rent virtual computers on which to run their own computer applications.
- [Khan Academy app for FirefoxOS](https://github.com/bbondy/khan-academy-fxos): Lead and primary developer for the Khan Academy app for Firefox OS. It handles both GET and POST /extensions update checks, serves Brave-managed components, redirects unsupported single-extension requests to Google's updater, and supports extension blacklisting. I sniffed Chromium's extension/component update HTTP API and flow and built this server implementation from that behavior.
